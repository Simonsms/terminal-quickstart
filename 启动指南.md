## 🎯 立即启动应用

### 方式一：开发模式（推荐）

```bash
cd terminal-quickstart
npm run tauri dev
```

**说明**：
- 首次运行会编译 Rust 代码（需要 5-10 分钟）
- 后续启动会很快（< 30 秒）
- 应用会自动打开窗口

### 方式二：仅前端预览

```bash
cd terminal-quickstart
npm run dev
```

**说明**：
- 在浏览器中预览前端界面
- 后端功能无法使用（Tauri 命令不可用）
- 适合快速查看 UI

## 🧪 功能测试建议

### 测试步骤

#### 1. 新建脚本
1. 点击「新建脚本」
2. 填写信息：
   ```
   名称: 测试脚本
   目录: C:\Windows\System32
   命令: echo Hello World && pause
   描述: 这是一个测试脚本
   ```
3. 点击「保存」
4. ✅ 检查：脚本卡片是否出现在列表中

#### 2. 启动脚本
1. 点击脚本卡片上的「启动」按钮
2. ✅ 检查：是否打开新的 CMD 窗口
3. ✅ 检查：是否输出 "Hello World"
4. ✅ 检查：是否显示成功提示

#### 3. 编辑脚本
1. 点击脚本卡片上的「编辑」图标
2. 修改脚本名称为 "测试脚本 - 已修改"
3. 点击「保存」
4. ✅ 检查：卡片是否更新

#### 4. 删除脚本
1. 点击脚本卡片上的「删除」图标（红色）
2. 确认删除
3. ✅ 检查：脚本是否从列表中移除

#### 5. 目录选择器
1. 新建脚本
2. 点击「浏览」按钮
3. 选择一个文件夹
4. ✅ 检查：路径是否填充到输入框

#### 6. 配置持久化
1. 添加几个脚本
2. 关闭应用
3. 重新启动应用
4. ✅ 检查：脚本是否仍然存在

## 🐛 常见问题

### 1. 首次启动很慢

**原因**: Rust 代码首次编译需要下载依赖  
**解决**: 耐心等待，仅第一次需要  
**时间**: 约 5-10 分钟

### 2. 找不到 Rust

**错误**: `cargo: command not found`  
**解决**: 安装 Rust 工具链  
```bash
# 访问 https://www.rust-lang.org/tools/install
# 下载并安装 rustup
```

### 3. 脚本启动失败

**可能原因**:
- 工作目录不存在
- 命令拼写错误
- 权限不足

**解决**:
- 检查目录路径
- 在终端手动测试命令
- 以管理员身份运行

### 4. 端口被占用

**错误**: `Port 1420 is already in use`  
**解决**: 关闭占用端口的程序，或修改 Vite 配置
